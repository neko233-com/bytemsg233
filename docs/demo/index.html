单文件 vs 散落文件

Protobuf — 多文件散落

user.proto
types.proto
common.proto
service.proto
agent 需要解析 import 链...

ByteMsg — 单文件 .bmsg

user.bmsg
一切在此，无需 import
// user.proto
import "types.proto";
import "common.proto";

message User {
  uint32 id = 1;
  string name = 2;
}
// user.bmsg — 一切在此
schema: bymsg/v1
package: com.example

enum UserType {
  ADMIN = 0
  USER = 1
}

message User {
  uint32 id = 1 // "ID" | "ID"
  string name = 2 // "名称" | "Name"
}

二进制编码原理

Varint 编码：小数字用更少字节，每字节 7 bit 数据 + 1 bit 标志位

值 = 1:
1 字节
值 = 127:
1 字节 (最大单字节值)
值 = 128:
2 字节 (溢出到双字节)
值 = 300:
2 字节
值 = 16384:
3 字节
值 = 100000:
3 字节
原理： 每字节最高位(MSB)为 1 表示后续还有字节，为 0 表示结束。 小数字 (0-127) 只需 1 字节，而 JSON 固定需要多个 ASCII 字符。

Wire Type 类型表

每个字段的 Field Header 由 Tag 和 Wire Type 组成

Wire Type值含义适用类型格式
Varint 0 可变长度整数 int32, int64, uint32, uint64, sint32, sint64, bool, enum Tag + Varint 值
Length-delimited 2 长度前缀数据 string, bytes,嵌套 message, list, map Tag + Length(Varint) + Data
注意： Wire Type 1 (64-bit) 和 Wire Type 5 (32-bit) 不在 ByteMsg 中使用。 ByteMsg 只用 Varint(0) 和 Length-delimited(2)，简化了解析逻辑。

Field Header 编码

Field Header = (Tag << 3) | WireType

Tag (字段编号)
Wire Type (类型标记)

示例: tag=1, wireType=0 (Varint) → (1 << 3) | 0 = 8 → Varint 编码: 08

Field 1 (Varint):
(1<<3)|0 = 8
Field 2 (String):
(2<<3)|2 = 18 = 0x12
Field 3 (Bytes):
(3<<3)|2 = 26 = 0x1A
Field 5 (Varint):
(5<<3)|0 = 40 = 0x28
优势： Field Header 将字段编号和类型压缩到单个 Varint 中。Tag 1-15 只需 1 字节 header， 比 Protobuf 的固定长度 encoding 更灵活，且向后兼容。

Zigzag 编码

将有符号整数映射为无符号整数，使 -1 和 1 都用 1 字节编码

原始值Zigzag 编码Varint 字节数说明
0010 → 0
-111-1 → 1
1211 → 2
-231-2 → 3
2412 → 4
-214748364842949672955最小 int32 → 最大 uint32

拖动滑块查看 Zigzag 编码效果:

-5 5
原始值: 0  →  Zigzag: 0    (1 字节) 
公式: (0 << 1) ^ (0 >> 63) = 0
为什么需要 Zigzag？ 普通 Varint 对负数 (如 -1 = 0xFFFFFFFFFFFFFFFF) 需要 10 字节。 Zigzag 将负数映射为小正数：-1 → 1, -2 → 3，使常见负数也只需 1 字节。

字符串 / 字节编码

字符串和字节数据使用 Length-delimited 格式：先写长度 (Varint)，再写数据 

// "Hello" 的编码
05           // 长度 = 5 (Varint)
48 65 6C 6C 6F // "Hello" (UTF-8)

// "Hi, 世界!" 的编码
0B           // 长度 = 11 (Varint)
48 69 2C 20 E4B896 E7958C 21 // UTF-8 bytes
// 嵌套 Message 也是 Length-delimited
// 先编码子 message 到临时 buffer
// 写入: Tag + Length + Buffer

12           // Field 2, WireType=2 (0x12)
08           // 子 message 长度 = 8
08 7B        // field 1: id = 123
12 05        // field 2: len = 5
48 65 6C 6C 6F // "Hello"

Map / List 嵌套编码

message UserProfile {
  uint32 id = 1;
  string name = 2;
  list<string> tags = 3;
  map<string, string> meta = 4;
}
// list<T> 编码:
// 1. 写入所有元素到 buffer
// 2. Header: Tag + WireType=2
// 3. Length: buffer 长度
// 4. Data: buffer 内容

// map<K,V> 编码:
// 等同 list<map_entry>
// 每个 entry: key(1) + value(2)
1A                    // Field 3, WireType=2 (Tag=3)
10                    // 总长度 = 16 字节
05                    // 元素数量 = 2 (list 长度)
05 61 64 6D 69 6E    // "admin" (长度 5 + 内容)
04 75 73 65 72       // "user"  (长度 4 + 内容)
22                    // Field 4, WireType=2 (Tag=4)
08                    // 总长度 = 8 字节
01                    // entry 数量 = 1
01 6B                 // key: "k" (长度 1 + 内容)
01 76                 // value: "v" (长度 1 + 内容)

CLI 使用

$ bytemsg233 init myproject

# 自动生成 myproject.bmsg:
schema: bymsg/v1
package: myproject

enum Status {
  ACTIVE = 0
  INACTIVE = 1
}

message Example {
  uint32 id = 1 // "ID" | "ID"
  string name = 2 // "名称" | "Name"
}

$ bytemsg233 compile user.bmsg \

    -o ./gen
